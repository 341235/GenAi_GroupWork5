# Python Docs RAG — Group Work 5

A question-answering chatbot that answers questions about Python, NumPy, Pandas, Matplotlib, scikit-learn, and Requests. It uses a local vector database for retrieval and the Groq Cloud API for answer generation.

---

## What this project does

You can ask questions like:
- *"How does the zip() function work?"*
- *"What is the difference between a list and a tuple?"*
- *"How do I handle exceptions in Python?"*
- *"How do I group a DataFrame by column in pandas?"*
- *"How do I create a scatter plot with matplotlib?"*
- *"How does train_test_split work in scikit-learn?"*

The app searches the documentation, finds the most relevant passages, and generates an answer based strictly on those passages. It remembers the conversation context so you can ask follow-up questions.

## Prerequisites — install these first

### 1. Python 3.10 or newer

### 2. Groq API Key (used for answer generation)
Sign up at [groq.com](https://groq.com) and create an API key. Or use the one provided my mail. 

Create a `.env` file in the project root with the following content:

GROQ_API_KEY=your-groq-api-key-here

If you want to run the evaluation script, install Ollama from [ollama.com](https://ollama.com) and pull the judge model:
```bash
ollama pull llama3.2
```
- **llama3.2** — used only during evaluation (scores the answers, small and fast)

## Setup (do this once)

Open a terminal in the project folder and run these commands one by one:

**Step 1 — Create a virtual environment** (keeps dependencies isolated from your system Python)
```bash
python3 -m venv venv
```

**Step 2 — Activate the virtual environment**

On Mac/Linux:
```bash
source venv/bin/activate
```
On Windows:
```bash
venv\Scripts\activate
```
You will see `(venv)` appear at the start of your terminal prompt — that means it is active.

**Step 3 — Install all dependencies**
```bash
pip install -r requirements.txt
```
This may take a few minutes the first time.

**Step 4 — Build the knowledge base**

This downloads the Python documentation pages and stores them locally as a searchable database:
```bash
python ingest.py
```

---

## Running the app

```bash
streamlit run app.py
```

A browser window will open automatically at `http://localhost:8501`. Type your Python question in the chat box and press Enter.

The first question after startup takes a bit longer as the embedding model and re-ranker load into memory — this is normal.

---

## Running the evaluation

The evaluation measures how well the system performs across 25 test questions using 5 metrics and saves a detailed report.

> **Note:** You need Ollama installed and running with the `llama3.2` model for evaluation.

```bash
python evaluate.py
```

This takes several minutes. Results are printed to the terminal and saved to `evaluation_results.csv`.

## Visualizing the embedding space

To generate an interactive 2D map of all document chunks stored in the database:

```bash
python visualize_embeddings.py
```

This creates `embedding_visualization.html` and opens it in your browser automatically. Each dot is one text chunk from the Python documentation. Chunks about similar topics cluster together. Hover over any dot to see which page it came from and a preview of the content.

The chunks are colour-coded by documentation category (Tutorial, Builtins, Data Structures, etc.) so you can see how well the different topic areas are separated in the embedding space.

> **Note:** You need to have run `python ingest.py` first to build the database.

## How it works

```
Your question + conversation history
     │
     ▼
┌──────────────────────────────────────────────┐
│           Multi-Query Generation             │
│  LLM rephrases the question 2 ways           │
│  → 3 query variants for broader coverage     │
└──────────────────────────────────────────────┘
     │
     ▼ (all 3 queries run in parallel)
┌──────────────────────────────────────────────┐
│    Smart Routing + Hybrid Retrieval          │
│  Routes to the right library (NumPy, Pandas  │
│  Matplotlib, sklearn, requests, stdlib, …)   │
│  BM25 keyword search (20%)                   │
│  + Semantic embedding search (80%)           │
│  → deduplicated candidate chunks             │
└──────────────────────────────────────────────┘
     │
     ▼
┌──────────────────────────────────────────────┐
│          Cross-Encoder Re-ranker             │
│  Scores each (question, chunk) pair          │
│  → keeps the 8 most relevant chunks          │
└──────────────────────────────────────────────┘
     │
     ▼
┌──────────────────────────────────────────────┐
│       Llama 3.1 8B (via Groq Cloud API)      │
│  Answers strictly from the retrieved chunks  │
│  Remembers previous turns in the chat        │
│  If the answer is not in the docs, it says:  │
│  "I couldn't find this in the Python docs."  │
└──────────────────────────────────────────────┘
     │
     ▼
  Answer + links to source pages
```

**Key design decisions:**

| Component | Choice | Why |
|---|---|---|
| Embedding model | `nomic-ai/nomic-embed-text-v1.5` | Uses separate task prefixes for indexing vs retrieval; stronger on technical documentation |
| Chunking (tutorial pages) | 1500 chars, 300 overlap | Preserves narrative context across longer explanations |
| Chunking (reference pages) | 900 chars, 180 overlap | Keeps each function description self-contained in one chunk |
| Retrieval | BM25 + semantic hybrid | Semantic understands meaning; BM25 ensures exact function names like `zip()` are always matched |
| Re-ranker | `cross-encoder/ms-marco-MiniLM-L-6-v2` | More accurate relevance scoring than embedding similarity alone |
| Generation model | Llama 3.1 8B (via Groq Cloud API) | Fast inference, no local GPU needed; better code understanding than Mistral |
| Multi-Query | 2 LLM-generated query variants | Broader retrieval coverage, all variants run in parallel |
| Conversation memory | Last N turn pairs in prompt | Enables follow-up questions without repeating context |

## Troubleshooting

**`GROQ_API_KEY` error or authentication failure**
Make sure you have a `.env` file in the project root with a valid `GROQ_API_KEY`. You can get one at [groq.com](https://groq.com).

**`ollama: command not found` (when running evaluation)**
Ollama is only needed for `evaluate.py`. Download it from [ollama.com](https://ollama.com) and make sure to launch the Ollama app.

**`Error: model 'llama3.2' not found` (when running evaluation)**
Run `ollama pull llama3.2` in a new terminal window.

**The first question takes a long time**
Normal behaviour — the embedding model and re-ranker load into memory on the first request. Subsequent questions are faster.

**`chroma_python_docs` folder is missing**
You skipped Step 4. Run `python ingest.py` to build the database.

**`(venv)` is not showing in my terminal**
You need to activate the virtual environment each time you open a new terminal window:
- Mac/Linux: `source venv/bin/activate`
- Windows: `venv\Scripts\activate`

**`ModuleNotFoundError` when running a script**
Make sure the virtual environment is active (see above) and that you ran `pip install -r requirements.txt`.
